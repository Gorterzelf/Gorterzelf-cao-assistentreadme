Project: CAO-Vraag Assistent

Een AI-prototype (MVP) voor het beantwoorden van CAO-vragen

Samenvatting

Dit document beschrijft een technisch prototype (MVP) van een AI-assistent die vragen over de CAO beantwoordt. De gekozen aanpak met LangChain in Python is veilig, uitbreidbaar en relatief eenvoudig te realiseren. We gebruiken open-source tools zoals FAISS, BM25 en HuggingFace-embeddings om maximale controle en flexibiliteit te behouden.

üí° Good to know: Wat is RAG?

Retrieval-Augmented Generation (RAG) is een AI-techniek waarbij het model eerst relevante informatie uit een bron (zoals een CAO-pdf) opzoekt en die context vervolgens gebruikt om een nauwkeurig antwoord te genereren. Dit zorgt voor feitelijke en betrouwbare antwoorden.

üõ†Ô∏è Stap 1: Installatie & Voorbereiding

  1. 1

    Installeer de basis (Python & VS Code)

    Download en installeer Python 3.10+ (vink "Add to PATH" aan) en Visual Studio Code.

  2. 2

    Project opzetten

    Maak een projectmap, open deze in VS Code en open de terminal (Beeld > Terminal).

    Activeer een virtuele omgeving (aanbevolen):

    python -m venv venv
.\venv\Scripts\activate
  3. 3

    Installeer Python Pakketten

    Voer dit commando uit in je terminal:

    pip install langchain langchain-openai python-dotenv langchain-community
  4. 4

    Maak een .env bestand

    Maak in je projectmap een bestand genaamd .env en voeg je Azure OpenAI-sleutels toe. Vervang xxxxx door je eigen keys.

    # Azure OpenAI settings
OPENAI_API_KEY=xxxxx
OPENAI_ENDPOINT=https://prd-oai-sw-svc.openai.azure.com/
OPENAI_API_VERSION=2024-02-15-preview
OPENAI_GPT_DEPLOYMENT=gpt-4
  5. 5

    Test je setup met main.py

    Maak een main.py bestand en plak de onderstaande code. Dit script test de verbinding met de AI-service.

    from langchain_openai import AzureChatOpenAI
from langchain.schema import SystemMessage, HumanMessage
from dotenv import load_dotenv
import os

# Laad .env-variabelen
load_dotenv()

# Ophalen van waarden
api_key = os.getenv("OPENAI_API_KEY")
api_base = os.getenv("OPENAI_ENDPOINT")
deployment = os.getenv("OPENAI_GPT_DEPLOYMENT")
version = os.getenv("OPENAI_API_VERSION")

# Initieer model
llm = AzureChatOpenAI(
    azure_deployment=deployment,
    azure_endpoint=api_base,
    api_key=api_key,
    api_version=version,
    temperature=0
)

# Simpele test
messages = [
    SystemMessage(content="Je bent een behulpzame assistent."),
    HumanMessage(content="Wat staat er in de CAO over overwerk?")
]

response = llm.invoke(messages)
print(response.content)

    Voer het script uit met python main.py. Als alles werkt, krijg je een antwoord van de AI!

üöÄ Publiceren op GitHub

Voordat je je project deelt, is het slim om je repository netjes voor te bereiden.

1. Maak een .gitignore bestand

Voorkom dat gevoelige bestanden (zoals .env) en onnodige mappen (zoals venv) worden ge√ºpload.

2. Genereer een requirements.txt

Leg de benodigde Python-pakketten vast zodat anderen je project makkelijk kunnen installeren. Gebruik pip freeze > requirements.txt.

3. Scan op veiligheidsrisico's

Controleer je gebruikte pakketten op bekende kwetsbaarheden met pip install pip-audit && pip-audit.

4. Gebruik een Personal Access Token (PAT)

Voor het pushen naar GitHub is een token veiliger dan je wachtwoord. Ga naar GitHub Tokens, genereer een nieuw token (Classic of Fine-grained) met `repo` (of `Contents: Read and write`) rechten. Kopieer het token. Wanneer de terminal om je wachtwoord vraagt, plak je het token.

Waarom een eigen AI-app bouwen?

  • Hogere nauwkeurigheid: De AI antwoordt op basis van j√≥uw documenten, niet op basis van algemene kennis. Dit voorkomt "hallucinaties".
  • Meerdere databronnen: Combineer eenvoudig een CAO, een beleidsregel en een handreiking in √©√©n assistent.
  • Slimmere zoektechnieken: Pas geavanceerde filters en zoekmethodes toe voor betere resultaten.
  • Tools en logica toevoegen: Laat de AI berekeningen uitvoeren of externe informatie opzoeken.
  • Grip op veiligheid: Jij bepaalt welke data wordt gebruikt, wat wordt gelogd en hoe het systeem reageert.
  • Schaalbaarheid: Eenvoudig te verbeteren en uit te rollen naar andere onderwerpen of sectoren.
