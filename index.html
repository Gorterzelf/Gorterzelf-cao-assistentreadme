üí° Wat is RAG?

Retrieval-Augmented Generation (RAG) is een AI-techniek waarbij het model eerst relevante informatie uit een bron (zoals een CAO-pdf) opzoekt en die context vervolgens gebruikt om een nauwkeurig antwoord te genereren. Dit zorgt voor feitelijke en betrouwbare antwoorden.

üöÄ Pro Tip: Hybrid Search

Voor de beste zoekresultaten, combineer keyword search (BM25) met semantic search (FAISS). Keywords vinden exacte termen (bv. "artikel 5.2"), terwijl semantiek de betekenis van een vraag begrijpt. Deze combinatie, 'Hybrid Search', geeft het beste van twee werelden en is een krachtige upgrade voor je RAG-systeem.

  4. 4

    Maak een .env bestand (ca. 5 min)

    Maak in je projectmap een bestand genaamd .env en voeg je Azure OpenAI-sleutels toe. Vervang xxxxx door je eigen keys. Heb je (nog) geen Azure OpenAI toegang? Informeer bij Martijn naar een API-sleutel en geef voor de zekerheid even je IP adres door zodat hij je kan whitelisten.

    # Azure OpenAI settings
OPENAI_API_KEY="xxxxx"
OPENAI_ENDPOINT="https://prd-oai-sw-svc.openai.azure.com/"
OPENAI_API_VERSION="2024-02-15-preview"
OPENAI_GPT_DEPLOYMENT="gpt-4"
  5. 5

    Test je setup met main.py (ca. 60 min, incl. oplossen van eventuele fouten)

    Maak een main.py bestand en plak de onderstaande code. Dit script test de verbinding met de AI-service.

    from langchain_openai import AzureChatOpenAI
from langchain.schema import SystemMessage, HumanMessage
from dotenv import load_dotenv
import os

# Laad .env-variabelen
load_dotenv()

# Ophalen van waarden
api_key = os.getenv("OPENAI_API_KEY")
api_base = os.getenv("OPENAI_ENDPOINT")
deployment = os.getenv("OPENAI_GPT_DEPLOYMENT")
version = os.getenv("OPENAI_API_VERSION")

# Valideer of alle variabelen zijn geladen
if not all([api_key, api_base, deployment, version]):
    print("Een of meer omgevingsvariabelen ontbreken. Controleer je .env bestand.")
else:
    # Initieer model
    llm = AzureChatOpenAI(
        azure_deployment=deployment,
        azure_endpoint=api_base,
        api_key=api_key,
        api_version=version,
        temperature=0
    )

    # Simpele test
    messages = [
        SystemMessage(content="Je bent een behulpzame assistent."),
        HumanMessage(content="Wat staat er in de CAO over overwerk?")
    ]

    try:
        response = llm.invoke(messages)
        print("AI Antwoord:", response.content)
    except Exception as e:
        print(f"Er is een fout opgetreden: {e}")

    Voer het script uit met python main.py in je terminal.

    Checkpoint: Zie je een antwoord van de AI in je terminal verschijnen? Mooi, dan heb je de basisverbinding succesvol opgezet! Gefeliciteerd, weer een mijlpaal behaald.

Neem voor deze stap ruim de tijd, ongeveer 60 minuten. Dit is inclusief de tijd die je de AI geeft om de code te genereren en de tijd om de app zelf te testen en eventuele fouten op te lossen.

Kopieer de onderstaande prompt en plak deze in ChatGPT (4o), Claude 3 Opus of Gemini 1.5 Pro. Vraag de AI om de Python-code voor je te schrijven.

Cre√´er een complete Python Streamlit-app voor mij in √©√©n bestand genaamd app.py. De app moet de volgende functionaliteiten hebben:

1.  **Titel:** De titel van de app is "CAO Vraag-Assistent".
2.  **PDF Upload:** Een gebruiker moet een PDF-bestand (de CAO) kunnen uploaden via een file uploader in de sidebar.
3.  **Verwerking:** Nadat de PDF is ge√ºpload, moet de app:
    a. De tekst uit de PDF extraheren.
    b. De tekst opsplitsen in behapbare 'chunks'.
    c. Deze chunks omzetten naar 'embeddings' met een HuggingFace-model (gebruik `HuggingFaceEmbeddings` met `sentence-transformers/all-MiniLM-L6-v2`).
    d. De embeddings opslaan in een FAISS vector store. Deze vector store moet in de sessie van de gebruiker bewaard blijven.
4.  **Vraag & Antwoord:**
    a. Een tekstinvoerveld waar de gebruiker een vraag kan stellen over de CAO.
    b. Wanneer een vraag wordt ingediend, moet de app de relevante chunks uit de vector store ophalen.
    c. Deze chunks, samen met de vraag, doorsturen naar een taalmodel (gebruik de `AzureChatOpenAI` setup).
    d. Het antwoord van het taalmodel weergeven aan de gebruiker.
5.  **Vereisten:** Gebruik de volgende Python-pakketten: `streamlit`, `pypdf`, `langchain`, `faiss-cpu`, `sentence-transformers`, `python-dotenv`, `langchain-openai`. Zorg dat de code de omgevingsvariabelen voor Azure OpenAI uit een `.env`-bestand laadt. De gehele logica moet in functies worden georganiseerd voor de duidelijkheid.

Sla de gegenereerde code op als app.py. Voer het daarna uit in je terminal met het commando: streamlit run app.py

1. Index Vooraf Bouwen

Maak een apart Python-script (bv. build_index.py) dat de CAO-PDF inleest, verwerkt en de FAISS-index lokaal opslaat. In je Streamlit-app hoef je deze index dan alleen nog maar te laden. Voordeel: Je app start veel sneller op, omdat het zware werk al is gedaan.

2. Implementeer Hybrid Search

Combineer de semantische zoekkracht van FAISS met de trefzekerheid van BM25 (keyword search). Gebruik de BM25Retriever uit LangChain naast je FAISS retriever om nog relevantere context te vinden. Voordeel: Grote verbetering in de nauwkeurigheid van de antwoorden.

3. Bronvermelding met Metadata

Zorg ervoor dat bij het inlezen van de PDF niet alleen de tekst, maar ook het paginanummer wordt opgeslagen als metadata bij elke chunk. Laat de AI in zijn antwoord altijd verwijzen naar de bronpagina's. Voordeel: De antwoorden worden controleerbaar en betrouwbaarder.

6. Begrippenlijst & Spiekbriefje

Om dit document voor iedereen toegankelijk te maken, volgt hier een overzicht van de belangrijkste termen en commando's.

Kernbegrippen (De Theorie)

RAG (Retrieval-Augmented Generation)
De techniek die we gebruiken. Eerst zoekt de AI relevante informatie op in een specifieke bron (de CAO), en gebruikt die kennis vervolgens om een feitelijk onderbouwd antwoord te formuleren.
Hybrid Search
De combinatie van zoeken op trefwoorden (zoals "artikel 5.2") en op betekenis (semantisch zoeken) voor de meest nauwkeurige resultaten.
Embeddings
Numerieke representaties (vectoren) van tekst. De AI gebruikt deze om de betekenis en context van woorden en zinnen te 'begrijpen' en met elkaar te vergelijken.
Vector Database (bv. FAISS)
Een speciaal type database, geoptimaliseerd om extreem snel te zoeken naar de meest relevante 'embeddings' die passen bij een gestelde vraag.

Technologie√´n & Frameworks (De Bouwblokken)

LangChain
Het 'brein' van onze applicatie. Een framework dat alle losse AI-onderdelen (zoals het taalmodel, de RAG-logica en de databronnen) aan elkaar knoopt tot een werkend geheel.
Streamlit
Een Python-bibliotheek waarmee we snel en eenvoudig een interactieve web-interface kunnen bouwen voor onze assistent, zonder complexe webdevelopment.
Python
De programmeertaal waarin we het hele project bouwen. Het is de meest populaire taal voor AI-toepassingen.
Azure OpenAI
De cloud-service van Microsoft die ons veilige en betrouwbare toegang geeft tot krachtige taalmodellen zoals GPT-4.
